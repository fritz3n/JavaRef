Keywords

A Keyword is a reserved word which has a special meaning and can´t be used in variable names.

Keyword Explanation
Control Flow
if Checks wether a boolean expression is true and, if it is, executes the following code block or instruction.
else Must follow an if and, if the preceding if didn´t execute its code block, executes the following code block or instruction.
for Executes a given instruction and executes the following code block or instruction while a given boolean expression is true, executing another given instruction each time.
while Executes the following code block or instruction while a given boolean expression evaluates to true. Checks the expression before execution.
do Must be followed by a code block and a while. Executes the code block while the expression specified in the while is true. Checks the expression after execution.
switch Must be followed by a code block containing cases. Executes the case matching a given case or, if no matching case is found, the default case.
case Specifies a case inside a switch.
default [case] Specifies the case that is executed if no other case matches. Optional.
break Breaks out of a for loop, while loop or switch case.
Exceptions
throw Throws an exception of type or subclass of Throwable.
throws Indicates that a method could throw a given type of exception.
try Must be followed by one or more catch blocks and/or a single finally block. Executes the following code block or instruction.
catch Is executed if an exception is generated inside the preceding try block and the type matches a given type. Prevents the Exception from being passed up the callstack. Exceptions inside the catch block behave like any other exception.
finally Executes after the preceding try block and catch blocks, independently of wether an exception was thrown inside the try block. Executes even if the method returns inside the try block.
Type Definition
class Defines a class. Defaults: private,
enum Defines an enum, which is a special type of class. Can be nested inside classes.
Inheritance
abstract [class] Indicates that a class can´t be instantiated. Often contains abstract methods. Subclasses typically implement all abstract methods and are not abstract, otherwise they must be declared abstract.
abstract [method] Marks methods that aren´t implemented. Abstract methods drop the curly brackets. A class containing abstract methods must itself be abstract.
final [class] Indicates that a class can´t be inherited.
final [method] Indicates that a method can´t be overwritten.
extends Indicates that a class extends (inherits) another class.
Access
public Indicates that the class, attribute or method is accessible by any other class.
default The default is that a class, attribute or method is accessible by any other class in the same package.
private Indicates that an attribute or method is only accessible within the containing class.
protected Indicates that an attribute is accessible by classes in the same package and subclasses
static Indicates that a method or attribute doesn´t need an object of the containing class to be called or modified.
